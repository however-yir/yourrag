# Contributing

Thanks for contributing to this project.

## Development Workflow

1. Fork and create a branch from the default branch.
2. Make focused changes with clear commit messages.
3. Run local checks (build/lint/tests) when available.
4. Open a PR with context, screenshots/logs if relevant, and risk notes.

## Commit Guidelines

- Use small, reviewable commits.
- Prefix optional: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`.

## Pull Request Checklist

- [ ] Scope is focused and backward-compatible, or compatibility impact is documented.
- [ ] README/docs are updated if behavior changed.
- [ ] Tests were added or existing tests still pass.
- [ ] Security-sensitive changes were reviewed.
